# Clarity Map

Water clarity reporting app for Israeli Mediterranean coast divers. Users submit reports from dive sites; data is visualized as colored markers and a heatmap on an interactive map.

**Live app**: [clarity-map.fly.dev](https://clarity-map.fly.dev) *(update this URL after first deploy)*

---

## Stack

| Layer | Tech |
|---|---|
| Backend | FastAPI (Python) |
| Frontend | Single HTML page + Leaflet.js |
| Storage | Google Sheets via gspread |
| Geo | Shapely + geopy |
| Deployment | Fly.io |

---

## Data Model

### Dive reports (`Sheet1`)

```
username | submitted_at | dive_datetime | clarity_m | beach | depth_m | lat | lon
```

- `submitted_at` — server-generated timestamp
- `dive_datetime` — user-set dive time (can be in the past, defaults to now)
- `clarity_m` — water clarity in meters (0–40, slider input)
- `depth_m` — dive depth in meters (auto-estimated or user-set)

### Weather forecasts (`windguru_forecasts` worksheet)

```
scrape_timestamp | forecast_datetime | wind_speed | gust_speed | wind_dir | swell_height | swell_period | swell_dir | station_id | station_name
```

Scraped hourly from [Windguru](https://www.windguru.cz) using two GFS models:

| Field | Source model | Windguru model ID |
|---|---|---|
| `wind_speed`, `gust_speed`, `wind_dir` | GFS 13km | 3 |
| `swell_height`, `swell_period`, `swell_dir` | GFS Wave Height (gfswh) | 84 |

Default station: **Tel-Aviv** (`WINDGURU_SPOT_ID=308`). Multiple stations can be added via a comma-separated `WINDGURU_SPOT_IDS` env var.

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/` | Map page |
| `GET` | `/admin/beaches` | Beach admin UI |
| `GET` | `/api/reports` | All reports as GeoJSON FeatureCollection |
| `POST` | `/api/report` | Submit a new dive report |
| `GET` | `/api/geo-info?lat=&lon=` | Sea check, depth estimate, nearest beach |
| `GET` | `/api/beaches` | List of dive sites |
| `POST` | `/api/beaches` | Save updated beach list |
| `GET` | `/api/grid` | Mean clarity per ~5km grid cell (GeoJSON) |
| `GET` | `/api/stats?hours=24` | Mean clarity for last N hours (24/48/168) |

Interactive API docs available at `/docs` when running locally.

## Local Development

### Prerequisites

- Python 3.12+
- [geos](https://libgeos.org/) system library: `brew install geos`

### Setup (first time)

```bash
# Clone and enter the project
git clone https://github.com/YOUR_USERNAME/clarity-map.git
cd clarity-map

# Create virtual environment
python -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Download Israeli coastline data
python prepare_data.py
```

### Credentials

The app needs two secrets:

1. **Google service account JSON** — place it at `.secrets/credentials.json`
   - Share the target Google Sheet with the service account email
2. **Google Sheet ID** — create a `.env` file:
   ```
   GOOGLE_SHEET_ID=your_sheet_id_here
   ```

Optionally add `GOOGLE_CREDS` to your shell for auto-loading:
```bash
echo 'export GOOGLE_CREDS="$(cat /path/to/.secrets/credentials.json)"' >> ~/.zshrc
```

### Run

```bash
source .venv/bin/activate
uvicorn main:app --reload
# open http://localhost:8000
```

On startup the app also starts the weather scraper scheduler (every 1 hour). Logs confirm: `[weather] Scheduler started — collecting every hour`.

## Deployment (Fly.io)

### First deploy

```bash
fly launch                  # creates fly.toml
fly secrets set GOOGLE_CREDS="$(cat .secrets/credentials.json)"
fly secrets set GOOGLE_SHEET_ID="your_sheet_id_here"
fly secrets set WINDGURU_SPOT_ID=308
fly deploy
```

### Auto-deploy via GitHub Actions

Every push to `main` triggers an automatic deploy. To enable this:

1. Get your Fly.io token: `fly auth token`
2. In GitHub: **Settings → Secrets → Actions → New repository secret**
   - Name: `FLY_API_TOKEN`
   - Value: paste the token

## Geo Data

### Coastline (`data/coast_israel.geojson`)

Generated by `python prepare_data.py`. Source: [Natural Earth 10m land polygons](https://www.naturalearthdata.com/downloads/10m-physical-vectors/), cropped to Israel bounding box. Used by Shapely to determine if a clicked point is in the sea.

### Depth estimation

`geo.estimate_depth(lat, lon)` — distance to coastline in km × 10, capped at 200m.

### Beach list (`data/beaches.json`)

~20 hardcoded Mediterranean dive sites with lat/lon. Editable via the admin UI at `/admin/beaches`. Mediterranean coast only (Red Sea / Eilat excluded).

---

## Beach List Maintenance

When you rename or reposition beaches, old reports in Google Sheets still reference the old name. Run the migration script to re-snap them to the nearest beach by coordinates:

```bash
# Dry run — shows what would change, touches nothing
python migrate_beaches.py

# Apply — updates the beach column in the sheet
python migrate_beaches.py --apply
```

This is also available directly from the `/admin/beaches` UI.

## Project Structure
